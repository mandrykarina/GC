    1) Переходим в папку mark_sweep - path/.../mark_sweep
    2) mkdir build ЕСЛИ НЕТ БИЛДА(СЕЙЧАС ЕСТЬ)
    3) cd build
    4) cmake ..
    5) make
    6) ./bin/main (запуск пользовательского интерфейса в консоли)


    # Про интерфейс #
- Первые 6 опций с дефолтными тестами и не уверен, что работают

- 7 - обычные тесты:
    1) Сначала инфа про параметры - надо нажать ENTER
    2) Далее выбор размера кучи (Heap) - от 10 мб до 1 гб(не менять, чтоб не сделать плохо компу)
    3) Выбор количества обьектов - лучше не выбирать слишком много, чтоб в случае чего не ждать. Оптимально до 2к для тестирования.
    4) Выбор размера каждого обьекта - лучше не выбирать 100% от текущего размера кучи, а то может упасть.
    5) Финальный вывод параметров.

- 8 - циклические тесты(все аналогично)

- 9 - каскадные тесты(все аналогично)

- 0 - ВЫХОД
